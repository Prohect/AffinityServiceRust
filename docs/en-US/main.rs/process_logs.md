# process_logs function (main.rs)

Processes `.find.log` files generated by the `-find` mode to discover new unmanaged processes. It scans a logs directory for find-log entries, filters out processes that are already covered by the loaded configuration or blacklist, and uses the Everything search engine CLI (`es.exe`) to resolve executable file paths. The results are written to an output text file for manual review and potential inclusion in the configuration.

## Syntax

```rust
fn process_logs(
    configs: &HashMap<u32, HashMap<String, ProcessConfig>>,
    blacklist: &[String],
    logs_path: Option<&str>,
    output_file: Option<&str>,
)
```

## Parameters

`configs`

A reference to the grade-keyed configuration map. The outer `HashMap<u32, ...>` is keyed by grade value, and the inner `HashMap<String, ProcessConfig>` is keyed by lowercase process name. Used to filter out processes that already have a configuration rule at any grade level.

`blacklist`

A slice of lowercase process names that should be excluded from the results. Processes appearing in the blacklist are treated as intentionally unmanaged and are not reported as new discoveries.

`logs_path`

An optional path to the directory containing `.find.log` files. When `None`, defaults to `"logs"` (relative to the current working directory). This corresponds to the `-in` CLI argument.

`output_file`

An optional path to the output file where results are written. When `None`, defaults to `"new_processes_results.txt"`. This corresponds to the `-out` CLI argument.

## Return value

This function does not return a value. Results are written to the specified output file. If the file write fails, an error message is logged to the console.

## Remarks

### Log file format

The function reads all files in the logs directory whose names end with `.find.log`. Each line in these files is expected to contain the substring `"find "` followed by the process name. The parser extracts the token immediately after `"find "` up to the next space (or end of line), and only retains entries ending in `.exe`. All process names are lowercased for case-insensitive comparison.

A typical `.find.log` line looks like:

```
[12:34:56]find notepad.exe default affinity
```

### Filtering pipeline

1. **Collect** â€” All unique `.exe` process names are gathered into a `HashSet` from every `.find.log` file in the logs directory.
2. **Filter by config** â€” Processes that appear in any grade level of `configs` are removed (they are already managed).
3. **Filter by blacklist** â€” Processes that appear in the blacklist are removed (they are intentionally excluded).
4. **Resolve paths** â€” For each remaining process, `es.exe` (the Everything command-line interface) is invoked with a regex search (`-r` flag and `-utf8-bom` output) to find the full file path on disk.

### Everything search integration

The function shells out to `es.exe` with the following arguments:

- `-utf8-bom` â€” Requests UTF-8 output with BOM.
- `-r` â€” Enables regex mode.
- `^<escaped_name>$` â€” Anchored regex matching the exact executable name (dots are escaped to literal `.`).

The stdout from `es.exe` is decoded using the system's active console output code page (obtained via `GetConsoleOutputCP`). For code page 936 (GBK), the `"gbk"` encoding label is used; otherwise, a `"windows-<codepage>"` label is constructed. If the encoding is not recognized, UTF-8 is used as a fallback. The BOM prefix (`0xEF 0xBB 0xBF`) is stripped before decoding if present.

If `es.exe` is not installed or fails, the output file records `"Not found, es failed"` for that process. If the search succeeds but returns no results, `"Not found, result empty"` is recorded.

### Output format

The output file contains one block per discovered process:

```
Process: example.exe
Found:
  C:\Program Files\Example\example.exe
  D:\Games\Example\example.exe
---
```

Or, if the executable was not found:

```
Process: unknown.exe
Not found, es failed
---
```

### Console mode

This function forces console output (`*get_use_console!() = true`) because it is intended for interactive use via the `-processlogs` CLI mode. The program exits immediately after this function returns.

### Typical CLI invocation

```
AffinityServiceRust.exe -processlogs -config config.ini -blacklist blacklist.txt -in logs -out new_processes_results.txt
```
