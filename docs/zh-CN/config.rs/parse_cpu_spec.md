# parse_cpu_spec 函数 (config.rs)

将 CPU 规格字符串解析为已排序、已去重的逻辑 CPU 索引向量。这是配置模块中使用的基础 CPU 规格解析器；它接受十六进制位掩码、包含范围和分号分隔的单个索引。所有其他感知 CPU 规格的函数最终都委托给 `parse_cpu_spec` 或使用其输出。

## 语法

```rust
pub fn parse_cpu_spec(s: &str) -> Vec<u32>
```

## 参数

| 参数 | 类型 | 描述 |
|------|------|------|
| `s` | `&str` | 支持以下描述的格式之一的 CPU 规格字符串。解析前会去除前导和尾随空白。 |

## 返回值

返回一个按升序排列且无重复项的 `Vec<u32>` 逻辑 CPU 索引。在以下情况下返回空向量：

- `s` 为空或仅包含空白。
- `s` 为 `"0"`（规范的"不更改"哨兵值）。
- `s` 为十六进制前缀（`0x` / `0X`）后跟无法解析为 `u64` 的值。

## 备注

### 支持的格式

| 格式 | 示例 | 结果 | 说明 |
|------|------|------|------|
| 空 / `"0"` | `""`、`"0"` | `[]` | 哨兵值，表示"不修改此设置"。 |
| 十六进制位掩码 | `"0xFF"`、`"0x0F"` | `[0,1,2,3,4,5,6,7]`、`[0,1,2,3]` | 旧格式；由于掩码以 `u64` 解析，限制为 64 个 CPU。前缀不区分大小写（`0x` 或 `0X`）。委托给 [mask_to_cpu_indices](mask_to_cpu_indices.md) 进行位提取。 |
| 包含范围 | `"0-7"` | `[0,1,2,3,4,5,6,7]` | 起始值和结束值均包含在内。如果结束值解析失败，则默认为起始值，生成单元素范围。 |
| 单个索引 | `"0;4;8"` | `[0,4,8]` | 分号分隔。每个部分周围的空白会被去除。非数字部分被静默跳过。 |
| 混合范围和索引 | `"0-3;8;12-15"` | `[0,1,2,3,8,12,13,14,15]` | 范围和单个索引可以自由组合。 |

### 去重和排序

由重叠范围或重复值产生的重复 CPU 索引在累积过程中被抑制（每个索引在插入前会检查现有向量）。最终向量在返回前按升序排列。

### 超过 64 个逻辑处理器的系统

范围和分号格式支持任意 `u32` 索引，使其适用于超过 64 个逻辑处理器的系统（例如，`"0-7;64-71"` 用于双处理器组配置）。十六进制位掩码格式限制为 64 位，应视为旧版兼容路径。

### 错误处理

`parse_cpu_spec` 有意采用宽容策略。无法解析的范围端点默认为 `0`，无法解析的单个索引被跳过，`0x` 后的无效十六进制值返回空向量。不会向调用方报告错误；带有用户可见错误消息的验证由更高级别的函数（如 [resolve_cpu_spec](resolve_cpu_spec.md)）负责。

### 示例

```text
parse_cpu_spec("0-3")       → [0, 1, 2, 3]
parse_cpu_spec("0;2;4")     → [0, 2, 4]
parse_cpu_spec("0x0F")      → [0, 1, 2, 3]
parse_cpu_spec("0-3;8;12")  → [0, 1, 2, 3, 8, 12]
parse_cpu_spec("0")         → []
parse_cpu_spec("")          → []
```
